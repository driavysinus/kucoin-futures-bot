# Парадигма Фен-Шуй — торговая логика сопровождения позиций

## Ключевой принцип

Вся логика привязана к **размеру начального стопа** (расстояние от цены входа до SL). Это единица измерения для всех действий: переводов в безубыток, порезов, углублений тейк-профита.

```
stop_size = abs(entry_price - sl_price)
```

---

## Поэтапная логика

### Исходные параметры при входе

```
entry_price  = цена входа
sl_price     = стоп-лосс
tp_price     = entry + 3 * stop_size (для buy) / entry - 3 * stop_size (для sell)
stop_size    = abs(entry_price - sl_price)
contracts    = начальный объём в контрактах
```

Тейк-профит автоматически рассчитывается как **3 стопа** от входа (соотношение R:R = 1:3 на старте).

---

### Этап 1 — Прошли 1 стоп в сторону T/P → Безубыток

**Условие:** цена достигла `entry ± 1 × stop_size`

**Действия:**
- Переносим SL на цену входа (безубыток)
- Новый `SL = entry_price` (+ минимальный отступ 1 тик)

**Результат:** риск по сделке = 0

```
Buy пример:  вход 0.6388, стоп 0.6368, stop_size=0.0020
             цена дошла до 0.6408 → SL переносим на 0.6389 (вход + 1 пункт)
```

---

### Этап 2 — Прошли 2 стопа → Первый порез + углубление SL и TP

**Условие:** цена достигла `entry ± 2 × stop_size`

**Действия:**
1. **Порез 50%** от начального объёма (округление вверх при нечётном)
   - Закрываем `ceil(contracts / 2)` контрактов по рынку
   - В рынке остаётся `contracts - ceil(contracts / 2)`
2. **SL** переносим на `entry + 1 × stop_size` (для buy) / `entry - 1 × stop_size` (для sell)
3. **TP** углубляем на `1 × stop_size` → новый TP = старый TP + stop_size

**Результат:** забрали риск, зафиксировали часть прибыли

```
Buy пример:  цена дошла до 0.6428
             Порез: 0.25 / 2 = 0.125 → округляем вверх → 0.13 лота режем
             Остаток: 0.12 лота
             Новый SL = 0.6388 + 0.0020 = 0.6408
             Новый TP = 0.6448 + 0.0020 = 0.6468
```

---

### Этап 3 — Прошли 3 стопа → Второй порез + углубление SL и TP

**Условие:** цена достигла `entry ± 3 × stop_size`

**Действия:**
1. **Порез 50%** от оставшегося объёма (округление вверх)
   - Закрываем `ceil(remaining / 2)` контрактов
2. **SL** переносим на `entry + 2 × stop_size`
3. **TP** углубляем на `1 × stop_size` → новый TP = предыдущий TP + stop_size

```
Buy пример:  цена дошла до 0.6448
             Порез: 0.12 / 2 = 0.06 лота
             Остаток: 0.06 лота
             Новый SL = 0.6408 + 0.0020 = 0.6428
             Новый TP = 0.6468 + 0.0020 = 0.6488
```

---

### Этап 4+ — Прошли N стопов (N ≥ 4) → Только углубление, без порезов

**Условие:** цена достигла `entry ± N × stop_size`

**Действия (на каждом новом стопе):**
1. **SL** углубляем на `1 × stop_size` → `entry + (N-1) × stop_size`
2. **TP** углубляем на `1 × stop_size`
3. **Порезов больше нет** — оставшийся объём сопровождается до SL или TP

```
Buy пример:  цена дошла до 0.6468 (4 стопа)
             Новый SL = 0.6428 + 0.0020 = 0.6448
             Новый TP = 0.6488 + 0.0020 = 0.6508
             
             Цена дошла до 0.6488 (5 стопов)
             Новый SL = 0.6448 + 0.0020 = 0.6468
             Новый TP = 0.6508 + 0.0020 = 0.6528
             
             ... и так далее, пока не сработает SL или TP
```

---

## Сводная таблица

| Стопов пройдено | Действие | SL | TP |
|:-:|---|---|---|
| 1 | Безубыток | entry + 1 тик | без изменений |
| 2 | Порез 50% от начального | entry + 1×stop | TP + 1×stop |
| 3 | Порез 50% от остатка | entry + 2×stop | TP + 1×stop |
| 4 | Только перенос | entry + 3×stop | TP + 1×stop |
| 5 | Только перенос | entry + 4×stop | TP + 1×stop |
| N | Только перенос | entry + (N-1)×stop | TP + 1×stop |

## Реализация — что хранится в Plan

```python
@dataclass
class Plan:
    symbol:         str
    side:           str        # buy / sell
    entry_price:    float      # цена входа (реальная, из позиции)
    contracts:      int        # начальный объём
    sl_price:       float      # текущий SL (обновляется)
    stop_size:      float      # abs(entry - initial_sl) — НЕ МЕНЯЕТСЯ
    leverage:       int
    
    # Текущее состояние
    remaining:      int = 0    # сколько контрактов осталось
    stops_passed:   int = 0    # сколько стопов пройдено
    current_tp:     float = 0  # текущий TP (обновляется)
    
    # ID ордеров на бирже
    entry_order_id: str = ""
    sl_order_id:    str = ""
    tp_order_id:    str = ""
    
    filled: bool = False       # вход исполнен
```

## Мониторинг цены

Этапы 1-4+ отслеживаются через WebSocket `price_update`. На каждом тике проверяем:

```python
distance = abs(current_price - entry_price)
current_stops = int(distance / stop_size)

if current_stops > plan.stops_passed:
    # Пройден новый уровень — выполняем действия этапа
```

Ордера SL и TP — реальные стоп-маркет ордера на бирже (reduceOnly). При каждом обновлении: отменяем старые → выставляем новые.

---

## Совместимость

- Вход: `/open`, `/stop`, `/alert` — все используют одну логику
- Параметры: `SYMBOL SIDE USDT PRICE SL [LEV]` — TRIG% больше не нужен
- TP рассчитывается автоматически: `entry ± 3 × stop_size`
- Порезы: `ceil()` — всегда в большую сторону
